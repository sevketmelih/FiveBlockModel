## 2. Data Architecture (Task 1)

### 2.1 Corpus and station selection

| Field | Detail |
| :--- | :--- |
| **Archive** | UCI ML Repository — PRSA Beijing Multi-Site Air Quality (2013–2017) |
| **Bundled stations** | 12 CSV files (Aotizhongxin, Changping, Dingling, Dongsi, Guanyuan, Gucheng, Huairou, Nongzhanguan, Shunyi, Tiantan, Wanliu, Wanshouxigong) |
| **Primary training site** | **Aotizhongxin** — $N = 35{,}064$ hourly records |
| **Features** | PM2.5, PM10, SO₂, NO₂, CO, O₃, temperature, pressure, dew point, precipitation, wind speed + encoded wind direction |

The pipeline downloads the official zip when missing, extracts to `PRSA_Data/`, and loads the Aotizhongxin CSV for the ablation pipeline.

### 2.2 Preprocessing contract (team baseline)

1. Build continuous `datetime` index; drop calendar/station identifier columns.
2. Fill missing sensor readings: `interpolate(method='linear')` → `ffill()` → `bfill()`.
3. One-hot encode `wd` (`drop_first=True`).
4. Reorder columns so **`PM2.5` is column 0**.
5. Chronological split at 70% / 15% / 15%.
6. `MinMaxScaler(feature_range=(0, 1))` — **fit on train only**, then transform val and test.
7. Sliding windows: input $\mathbf{X} \in \mathbb{R}^{24 \times D}$, target $y_{T+1}$ = scaled PM2.5 at the next hour.

This contract was preserved by Tasks 2–4; later work added numeric-only interpolation (pandas ≥ 2.1), `test_target_hours`, DAE training noise, and BiLSTM/residual blocks without changing the split or scaler rules.

### 2.3 Window shapes (representative run)

After preprocessing on Aotizhongxin:

## 3. Baseline Model & Ablation (Initial `main.py`)

Commit `6a1bc9f` delivered the first end-to-end training script consumed and extended by Student **210911026** (Task 2).

### 3.1 Five-block architecture (v1)

| Block | Layer(s) | Role |
| :---: | :--- | :--- |
| **1** | TimeDistributed DAE (`latent_dim=8`) | Sequence reconstruction + denoised latent feed-forward |
| **2** | Conv1D(64, k=3) + BN + MaxPool1D(2) | Local temporal features |
| **3** | LSTM(64, `return_sequences=True`) + BN | Recurrent temporal memory |
| **4** | `SimpleAttention` or `GlobalAveragePooling1D` | Temporal weighting / pooling fallback |
| **5** | Dense(64) → Dense(32) → Dense(1) | Forecast head (dropout 0.3 / 0.2, L2 $10^{-4}$) |

## 4. Experimental Results (Commit `6a1bc9f`, Clean Test Only)

*Source: `outputs/ablation_metrics_comparison.csv` at initial delivery. No noisy-test protocol in this commit.*

| Scenario | MSE (µg/m³)² | MAE (µg/m³) | R² |
| :--- | :---: | :---: | :---: |
| **Model A (Full — 5 Blocks)** | 1726.84 | 27.18 | 0.8008 |
| **Model B (No Autoencoder)** | **1167.79** | **21.43** | **0.8653** |
| **Model C (No Attention)** | 1214.96 | 22.57 | 0.8598 |
| **Model D (Base CNN+LSTM)** | 2501.36 | 32.08 | 0.7115 |

### 4.1 Key findings documented in README §5.2

- **CNN + LSTM + Attention** raise $R^2$ from **0.7115** (Model D) to **0.8653** (Model B).
- **Self-attention** (B vs. C) improves $R^2$ by ~0.005 on clean data.
- **DAE trade-off:** Model B beats Model A on clean test ($R^2$ 0.865 vs. 0.801) because joint reconstruction ($\beta=0.2$) regularizes the encoder across all meteorological channels — README frames this as expected robustness under corrupted sensors (validated later by Task 2 noise ablation).
