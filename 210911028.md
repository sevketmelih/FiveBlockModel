## 2. Design Philosophy: From Assignment Plots to Paper-Ready Figures

Prior iterations saved figures without explicit `dpi=300` on `savefig`, mislabeled a **time-series line chart** as a scatter plot, and used a raw `imshow` heatmap without hour-of-day semantics. Task 4 introduces a dedicated module that:

1. Centralizes **matplotlib + seaborn** styling (`apply_publication_style()`).
2. Enforces **300 DPI**, `bbox_inches='tight'`, and white backgrounds on every export.
3. Separates **semantic figure types** (scatter vs. time-series vs. heatmap vs. bar).
4. Maps self-attention weights to **clock hours** using test-set target timestamps and CNN pooling geometry.

### 3.2 Integration in `main.py`

- `download_and_preprocess_data()` returns `test_target_hours` — hour-of-day (0–23) for each test forecast target.
- Inline matplotlib blocks replaced by `generate_all_publication_figures(...)`.
- `save_training_histories(histories)` called before figure export so `regenerate_all_figures.py` can plot loss curves without retraining.
- Attention analytics receive both `model_a_attention_weights` and `test_target_hours`.
- `regenerate_all_figures.py` uses `load_production_hyperparameters()` so checkpoint shapes match Optuna-tuned architecture.

### 3.3 Attention → clock hour mapping

After Block 2 (`MaxPool1D(pool_size=2)`), the 24-hour window compresses to **12 pooled steps**. For each test sample with target hour $h^\*$:

$$\text{clock\_hour}(p) = (h^\* - \text{lag}(p)) \bmod 24, \quad \text{lag}(p) = 24 - 2p - 1$$

Mean attention is aggregated over all test samples per clock hour. **Rush-hour bands** (07:00–09:00, 17:00–20:00) are shaded on `attention_hour_of_day.png`; `attention_rush_hour_comparison.png` reports the rush/off-peak ratio.

### 3.4 Hyperparameter tuning plots (Task 3 alignment)

| **D (Base)** | 2433.63 | 33.11 | 0.7193 |

### Table 3 — Robustness (ΔR² = Noisy − Clean)

| Model | R² (Clean) | R² (Noisy) | ΔR² |
| :--- | :---: | :---: | :---: |
| **A (Full)** | 0.8690 | 0.8388 | −0.0302 |
| **B (No DAE)** | 0.8631 | 0.8046 | −0.0585 |
| **C (No Attn)** | 0.8499 | 0.8432 | −0.0067 |
| **D (Base)** | 0.8719 | 0.7193 | −0.1526 |

### Table 4 — DAE Contribution (A vs. B)

| Comparison | Clean ΔR² (A − B) | Noisy ΔR² (A − B) |
| :--- | :---: | :---: |
| DAE advantage | +0.0059 | **+0.0343** |

**Visualization insight:** Model A’s R² degradation under noise (−0.030) is roughly **half** that of Model B (−0.059) and Model D (−0.153), which the grouped bar chart (`ablation_clean_vs_noisy_r2.png`) communicates for non-technical readers.
