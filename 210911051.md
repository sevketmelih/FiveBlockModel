## 2. Implementation Overview

### 2.1 Library and Sampler

[Optuna](https://optuna.org/) was added as a project dependency (`optuna>=3.5.0`). The **Tree-structured Parzen Estimator (TPE)** sampler was used, seeded at `RANDOM_SEED = 42` to make the search reproducible.

TPE is a Bayesian optimization method that builds probabilistic models of the objective function. Unlike a grid search, it learns from earlier trials to focus sampling on promising regions of the hyperparameter space.

### 2.2 Search Space

| Hyperparameter | Type | Candidates |
| :--- | :---: | :--- |
| `learning_rate` | categorical | `1e-2`, `1e-3`, `1e-4` |
| `dropout_rate` | categorical | `0.2`, `0.3`, `0.5` |
| `bilstm_units` | categorical | `32`, `64`, `128` |

`dropout_rate` is applied to both `dropout_1` and `dropout_2` layers of the forecast head simultaneously, keeping the search space manageable (27 possible combinations).

### 2.3 Objective Function

Each trial:
1. Samples one combination of the three hyperparameters.
2. Builds the **full 5-Block model** (DAE + CNN + BiLSTM + Residual + Attention) with those values via `build_hybrid_model()`.
3. Injects **DAE denoising noise** on auxiliary sensors (`TRAIN_DAE_NOISE_STD=0.04`, PM2.5 untouched) — same as `main.py` ablation training.
4. Trains for a maximum of **12 epochs** with `EarlyStopping(patience=5)` on the validation loss.
5. Returns the **minimum validation loss** achieved across all epochs.

The DAE compound loss (forecast MSE + reconstruction MSE × β=0.05) is used as the objective, consistent with the main training protocol in `main.py`.

### 2.4 Data Protocol

Data is loaded **once** before the Optuna study begins and reused across all trials, avoiding redundant I/O. The same chronological 70/15/15 split and `MinMaxScaler` (fit on train only) from `main.py` are used to prevent data leakage.

## 3. Bug Fix: Pandas Compatibility (`main.py`)

During integration, a compatibility issue was found in `main.py`:

```python
# Before (fails on pandas >= 2.1 with string dtype columns)
df = df.interpolate(method='linear').ffill().bfill()

# After (interpolates numeric columns only; string columns filled separately)
num_cols = df.select_dtypes(include="number").columns
df[num_cols] = df[num_cols].interpolate(method="linear").ffill().bfill()
df = df.ffill().bfill()
```

The wind direction column (`wd`) is a string/categorical column that must not be passed to `interpolate()`. Newer versions of pandas raise a `TypeError` in this case. The fix isolates numeric columns for interpolation and uses `ffill/bfill` for all remaining columns.

### 4.3 Key Finding: Hand-Picked Defaults Were Suboptimal

The original `main.py` used `learning_rate = 1e-3` (the Adam default). Optuna discovered that **`learning_rate = 1e-2`** consistently outperforms it across multiple trials:

| Configuration | val_loss | Improvement |
| :--- | :---: | :---: |
| Default (`lr=1e-3`, `dropout=0.2`, `bilstm=64`) | ~0.038 | baseline |
| **Optimal (`lr=1e-2`, `dropout=0.5`, `bilstm=64`)** | **0.009** | **~4.2× lower** |

**Sampler convergence:** From Trial 5 onward, the TPE sampler consistently sampled `lr=1e-2` with `dropout=0.5`, demonstrating that the Bayesian search identified and exploited this region without exhaustive enumeration.
